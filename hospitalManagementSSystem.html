Project Overview

Why this project?

Healthcare systems run on complexity — patient records, staff rosters, appointment queues, billing, and clinical outcomes all need to flow through a single coherent interface. This project was an exercise in designing and building that interface from scratch using React and Tailwind CSS.

The goal was to simulate what a real hospital IT department might commission: a desktop-first dashboard that clinical administrators can operate without confusion, even under pressure.

Problem Statement

The pain of fragmented tools

Most hospital management UIs are either legacy desktop software with poor ergonomics, or fragmented web tools that don't talk to each other. Clinical administrators end up context-switching constantly — opening one tab for patient data, another for staff schedules, a third for analytics.

  • No unified view of patient status across wards
  • Staff scheduling managed in spreadsheets
  • Analytics siloed from operational data
  • No clear hierarchy for urgent vs. routine tasks

Challenges & Solutions

What I had to solve

🔴 Challenge

Desktop-first layout breaking at tablet widths without ugly scrollbars or clipping.

🟢 Solution

Used CSS Grid with minmax() and collapsible sidebar panels controlled by a toggle state in React.

🔴 Challenge

Charts rendering slowly on initial paint due to large data arrays.

🟢 Solution

Lazy-loaded chart components with React.lazy and Suspense, pre-computed chart data outside render cycles.

🔴 Challenge

Keeping search state in sync across multiple modules without prop drilling.

🟢 Solution

Lifted search state to a context provider — one source of truth consumed anywhere in the tree.

Lessons Learned

What this taught me

  • Information architecture first. The sidebar navigation structure had to be right before any component was written. Getting that wrong early costs many refactors.
  • Dark UIs demand discipline. Contrast ratios on dark backgrounds are easy to break — Tailwind's colour scale needs careful selection at each opacity level.
  • Desktop-first is different from mobile-last. Starting wide and compressing down requires explicit breakpoint strategies, not just reverting defaults.
  • Data visualisation is its own design discipline. Recharts is flexible but its axis labels, legend placement, and tooltip styling all need intentional design decisions.
